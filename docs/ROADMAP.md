# Roadmap

Now / Next / Later. No dates, no promises - direction, honestly stated. Items
move only when someone does the work.

The governing delivery order is the
[CURATIONS.DEV Vibe Coding Community PRD](PRD-curations-community.md). The
[Project Evidence Registry PRD](PRD-project-evidence-registry.md) governs its
subordinate GitHub intake and trust layer. Feature PRDs may not reorder either
contract.

## Now

- Draft governing conversation-first community contract in PR #12.
- Approved fixture-driven Community Board, Projects, Stacks, and Community
  Library discovery shell live at `curations.dev`.
- Illustrative fixture activity and noindexed fixture Project details; no claim
  of an active community or measured outcomes.
- Published Astro Board surface on Azure Static Web Apps, with the Azure
  Container Apps gateway, Cosmos DB, GitHub identity, and bounded optional Azure
  AI Foundry integration.
- Curated software directory, versioned Cookbooks, prompt and workflow libraries,
  taxonomy, schemas, offline CLI, quality model, and contribution guide.
- **Use My Copilot** and **Run in My Terminal** remain separate user-funded
  execution lanes with no silent CURATIONS-funded fallback.

## Next - Contract and visual proof

- Approve the governing Community PRD and retain the Evidence Registry as its
  trust layer.
- Complete the fixture-only builder journey:
  - **I have a working plan**
  - **Help me create one from my Project**
  - private Project Review
  - editable human feedback request
  - exact public preview and explicit consent
  - minimal profile context
- Preserve the existing conversation rows, red GitHub Project backlinks, linked
  Stack pills, one tab-controlled homepage feed, and contextual right rails.
- Validate that a newcomer understands the Project and requested feedback before
  persistence or model calls are added.

## Next - Trust gate

- Make public agent invocation explicit and default-off.
- Replace overloaded "verified" language with declaration, observation, consent,
  review, and freshness.
- Enforce repository-owner match for public-Project submission.
- Surface or reject fork, archived, stale, unavailable, and revoked states.
- Add builder revocation and maintainer moderation.
- Keep **Propose a tool** separate from **Submit your Project**.
- Publish inspectable Project evidence methodology.
- Retire or hard-gate the legacy Cloudflare production deployment lane.

## Next - Thin Project loop

- Implement the existing-plan path for one invited builder, then expand to five.
- Add only the Project, plan revision, conversation, review, evidence snapshot,
  consent, moderation, and revocation records needed by that path.
- Pin plan and evidence reads to one repository commit.
- Require a private review, exact public preview, explicit consent, and manual
  publication review.
- Record accepted, adapted, declined, or deferred advice only through an explicit
  builder action.
- Test the governing PRD's falsification criteria before expansion.

## Later - Earned after the pilot

- Reverse-plan generation from explicitly allowlisted public repository material.
- Terminal handoff after the Project Review contract is stable.
- Reviewed, version-pinned skill and Cookbook matching with explicit outcomes.
- Refresh history, organization/collaborator consent, and additional evidence
  rules.
- Human comments and peer review after moderation controls exist.
- Profiles, following, human-mediated mentoring, and broader Project collections.
- Transparent trends only after genuine submissions and outcomes exist.
- CURATIONS Credits only as a separately approved hosted-execution product.

## What we will not do

- Source or crawl repositories their builders did not submit.
- Claim that a PRD or configuration file proves production use or quality.
- Rank Projects or tools as "best" from counts or votes.
- Read `.env`, secrets, terminal history, browser cookies, or unrelated files.
- Auto-install, auto-update, auto-activate, or auto-run external skills.
- Spend CURATIONS model credits for a user who selected **Use My Copilot** or
  **Run in My Terminal**.
- Collect covert product analytics or behavioral profiles.
- Treat available AI capacity as permission for invisible or unbounded use.
